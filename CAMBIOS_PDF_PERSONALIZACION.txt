CAMBIOS REALIZADOS EN LA PERSONALIZACIÓN DE PLANTILLAS PDF (2025-08-25)
============================================================

1. Sincronización Frontend-Backend de Plantillas PDF
----------------------------------------------------
- El frontend ahora envía TODA la configuración personalizada (posiciones, autores, firmas, colores, tamaño de fuente, etc.) al backend cuando se aplica la plantilla personalizada.
- El backend guarda la plantilla activa y su configuración en `config/pdfTemplateConfig.json`.
- El PDF generado respeta la plantilla seleccionada y las posiciones/configuración enviadas.

2. Corrección de Estado Activo de Plantilla
-------------------------------------------
- El selector visual y la preview en el frontend siempre muestran la plantilla activa real.
- Al aplicar una plantilla, el backend la guarda y el frontend la refleja correctamente.
- Ya no se queda fija en "Elegante"; la plantilla activa es la que el usuario selecciona y aplica.

3. Preview y PDF Real Iguales
-----------------------------
- Se corrigió el flujo para que la preview y el PDF generado sean lo más parecidos posible, usando los mismos datos de configuración.
- El backend renderiza los campos según las posiciones y estilos enviados desde el frontend.

4. Mejoras de UX
----------------
- El botón de aplicar muestra feedback visual y notificaciones.
- El usuario puede mover campos en la preview y esos cambios se reflejan en el PDF real.

5. Seguridad y Roles
--------------------
- Solo el usuario con rol "owner" puede cambiar la plantilla global (por seguridad en producción).

6. Archivos y rutas clave
-------------------------
- Frontend principal: `scripts/frontend/profile.frontend.js`
- Configuración global: `config/pdfTemplateConfig.json`
- Backend de plantillas: `scripts/controllers/pdfTemplate.controller.js`, `scripts/routes/pdfTemplate.routes.js`
- Render PDF: `scripts/utils/pdf.js`

NOTA: Si retomas el proyecto o lo entregas a otra IA, revisa que el frontend y backend usen la misma estructura de configuración y que el endpoint `/api/pdf-template/config` esté correctamente protegido y sincronizado.

Fin de cambios.
